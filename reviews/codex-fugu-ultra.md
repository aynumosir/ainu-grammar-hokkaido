**P0 (Must-Fix Before Authoring)**
* **P0 [Chapter Outline - Ordering]:** The current macro-outline places basic clause structure (Part XV) and nominalization (Part XVI) after the complex TAM and evidential operators (Parts XII–XIV) that structurally depend on them. **Fix:** Reorder the outline so that fundamental syntax and nominalization logically and numerically precede the operator paradigms.
* **P0 [Chapter Outline - Granularity & Gaps]:** The 198-chapter outline exceeds the 180-chapter target with redundant splits (e.g., separating coda phonotactics from loanword adaptation, or splitting the 4th person across four chapters) while lacking necessary overview chapters. **Fix:** Reopen the TOC to execute the exact chapter merges proposed in the coherence review and insert the missing TAM, valency, and NP-internal structure overviews, trimming the total to ~175 chapters.
* **P0 [Editorial Methodology]:** Mandating a novel "original analysis" in every single leaf chapter (Step vi) encourages editorial overreach and undermines the stability of a reliable reference work. **Fix:** Replace this mandate with explicit evidence labels (e.g., consensus, contested, original proposal) and strictly limit original theoretical re-analysis to the five explicitly named "theory-magnet" domains.
* **P0 [Source Apparatus & Scope]:** The mandate to "cite every Ainu material ever" conflates background source registration with actual evidentiary citation, while publishing thousands of fully glossed sentences from external editions carries severe copyright exposure. **Fix:** Create a strict distinction between the exhaustive registered bibliography and the actually cited materials, and define formal fair-use thresholds and permission requirements for the extended texts planned in Part XXII.
* **P0 [Execution Roadmap]:** The parallel authoring model risks severe terminological and analytic drift if multiple agents simultaneously draft prose before the baseline analysis of core phenomena is settled. **Fix:** Insert a mandatory serial human-editorial sign-off gate on the claims-matrices (Step ii) for foundational chapters (e.g., person-marking, valency, nominalization) before authorizing concurrent prose drafting for dependent chapters.
* **P0 [Scope & Integrity]:** Treating modern revitalization 'neo-Ainu' as a synchronic variety throughout the text risks contaminating the traditional historical descriptive baseline with artificially coined constructions. **Fix:** Confine the description of revitalization norms and neologisms strictly to the sociolinguistic overview (Chapter 4) and clearly delineated contrast boxes, excluding them from all baseline corpus frequency counts.

**P1 (Should-Fix)**
* **P1 [Technical Architecture - Build Scaling]:** Running heavy compositional gloss validation and dynamic Katakana script conversion inside thousands of `<Ex>` components during SvelteKit prerendering will create massive build bottlenecks and out-of-memory risks. **Fix:** Shift example validation, script conversion, and stable UUID assignment to an offline pre-build script that emits a clean, static JSON artifact for the UI components to consume.
* **P1 [Technical Architecture - Cross-references]:** The proposed `<ExRef>` component relies on parsing the compiled output to generate `example-index.json`, introducing a fragile two-pass compilation dependency. **Fix:** Assign stable, static IDs directly in the `examples.jsonl` files to allow robust, single-pass cross-reference resolution that fails CI on dangling targets.
* **P1 [Source Apparatus]:** Foundational must-cite works (Batchelor, Patrie, Asai, Hattori) are mentioned in the prose but lack attached citation keys, and the citation key formats themselves are wildly inconsistent (e.g., `authorYEAR` vs `authorYEAR-suffix`). **Fix:** Enforce a single canonical citation key scheme, build a direct key-to-db-slug crosswalk, and formally attach all required historical and typological framework keys to their respective chapters before authoring.
* **P1 [Editorial Methodology]:** The corpus-quantified layer under-specifies how it will handle duplicated editions, genre imbalances, and transcription noise when computing frequencies. **Fix:** Draft a rigorous statistical protocol prior to authoring that defines countable units, deduplication, dialect weighting, and manual validation thresholds for every numerical claim.
* **P1 [Execution Roadmap]:** The execution roadmap contains hardcoded, incorrect chapter numbers (e.g., "Ch 16 consonant-inventory" which is actually 15) and stale wave chapter counts that contradict the finalized outline. **Fix:** Strip all hardcoded chapter numbers from the roadmap in favor of stable slugs, run a TOC-validation audit, and dynamically regenerate the wave groupings based on the newly trimmed TOC.

**P2 (Nice-to-Have)**
* **P2 [Technical Architecture - Maps & Search]:** Developing bespoke vector maps for Hokkaido and tuning Pagefind for a multi-script, facet-heavy grammar will become disproportionate technical sinks. **Fix:** Descope cartography to a single tile-based MapLibre map, and configure Pagefind to emit specific metadata facets (part, dialect, genre, script) to support complex linguistic queries.
* **P2 [Chapter Outline - Texts]:** Part XXII bundles two texts (yukar and kamuy yukar) into a single chapter (189), breaking the "one text per chapter" consistency seen in the rest of the part. **Fix:** Split Chapter 189 into two separate text chapters if permissions and length allow, to maintain structural uniformity.
* **P2 [Scope & Integrity]:** The "comprehensiveness, not priority" honesty rule is an excellent guideline but relies entirely on manual review to catch priority claims. **Fix:** Implement a build-time linting rule that flags phrases like "first," "unique," or "never before," requiring an explicit source-audit override to pass.

**Overall Verdict**
The proposed build plan is an exceptionally rigorous, well-conceived blueprint that correctly identifies critical gaps in Ainu linguistics and deploys a state-of-the-art methodology to address them. However, its immense ambition threatens its execution: the mandate for universal novel analysis, the sheer volume of fragmented/misordered chapters, and heavy build-time component validation introduce significant scalability and quality risks. If the editors aggressively trim the chapter count via the proposed merges, reorganize the outline to put fundamental syntax before operator paradigms, limit original analysis to key domains, and optimize the SvelteKit build pipeline for static data ingestion, this project is fully ready to execute. The single most important improvement is the immediate reorganization of the macro-outline (moving basic clause structure before TAM/Evidentiality) to ensure logical dependency coherence for both the parallel authoring agents and the final readers.
