**Diagnosis**

The “weird whites” are mostly a surface-system problem. Tokens exist in [src/app.css](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/app.css:23), but the UI uses `canvas` and `surface` inconsistently: header/search/chart are white surfaces, while the main reading area is unframed canvas text. On light mode that reads like accidental white strips/cards on a nearly white page, not a designed book system.

**P0**

- Problem: No coherent page/surface hierarchy. Fix: introduce a two-tier rule: `body/html = bg-canvas`; header/sidebar/article/chart/search popover = `bg-surface border border-hairline`; add square elevation tokens such as `--shadow-raised: 0 1px 2px rgb(15 23 42 / .06), 0 0 0 1px var(--color-border)` and dark equivalent, then use them only for sticky chrome/popovers, not every block.
- Problem: Reading column is just `.g-main max-width: 46rem` on canvas, causing unfinished open whitespace in [grammar layout](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/routes/grammar/+layout.svelte:173). Fix: set `.g-main { max-width: 68ch; margin-inline: auto; }` and wrap `.grammar-chapter` / `.grammar-cover` in `background: var(--color-surface); border: 1px solid var(--color-border); padding: clamp(1.5rem, 4vw, 3rem);` with square corners.
- Problem: Sidebar is structurally naked on desktop but bordered on mobile only in [grammar layout](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/routes/grammar/+layout.svelte:77). Fix: make `.g-sidebar` a first-class navigation surface on all viewports: `background: var(--color-surface); border: 1px solid var(--color-border); padding: 1rem; box-shadow: var(--shadow-raised);`.
- Problem: Header/footer do not share the same chrome language; header is `bg-surface`, footer is transparent in [SiteHeader](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/lib/components/SiteHeader.svelte:26) and [SiteFooter](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/lib/components/SiteFooter.svelte:51). Fix: use `bg-surface/ border-hairline` for both, same max-width as grammar shell, same `font-sans text-sm`, and add active nav state for `/grammar`.
- Problem: Prose links are stripped to color-only until hover in [grammar layout](/home/mkpoli/projects/Ainu/ainu-grammar-hokkaido/src/routes/grammar/+layout.svelte:270). Fix: restore scholarly link affordance: `.g-main :global(p a, li a, td a, .ref, .xr) { color: var(--color-accent); text-decoration-line: underline; text-decoration-thickness: .06em; text-underline-offset: .18em; text-decoration-color: color-mix(in srgb, var(--color-accent) 45%, transparent); }`, with stronger decoration on hover/focus.
